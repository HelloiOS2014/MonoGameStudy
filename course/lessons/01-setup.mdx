## Goal

Prepare the machine for .NET 10 and MonoGame DesktopGL, then verify that this repo can create and run the same project shape used by the experiments.

## Run

```bash
./tools/check-env.sh
```

## Observe

The script reports macOS readiness, the active .NET SDK, `global.json`, MonoGame template availability, and MGCB-related tooling expectations. Treat any missing tool as a setup blocker before running experiments.

## Expected Visual State

No DesktopGL window opens in setup. The expected state is terminal output from `./tools/check-env.sh` showing macOS, `global.json`, .NET 10, and the `mgdesktopgl` template check.

## Key Files

- `global.json`
- `tools/check-env.sh`
- `GameDemo.sln`

## Walkthrough

`global.json` pins the repo to .NET 10. The checked-in value is `10.0.100` with `rollForward` set to `latestFeature`, so a newer installed 10.0 feature band can be selected.

`tools/check-env.sh` is the repository-level readiness check. It verifies the current OS target, confirms `global.json` exists, checks that the selected SDK is .NET 10, and confirms the `mgdesktopgl` template is installed. Run it from the repository root before opening a MonoGame window.

## Common Failures

- `dotnet` is missing from `PATH`: install the pinned SDK and open a fresh shell.
- Wrong SDK: make sure `global.json` is at the repo root and a .NET 10 SDK is installed.
- MonoGame templates are missing: install templates with `dotnet new install MonoGame.Templates.CSharp`.
- MGCB tooling is missing or stale: restore local tools in the experiment directory that owns the content pipeline.
