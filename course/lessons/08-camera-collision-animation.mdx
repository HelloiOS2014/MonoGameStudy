## Goal

Add camera transforms, collision checks, and simple animation state while keeping game logic separate from MonoGame drawing.

## Run

```bash
env E06_SMOKE_EXIT_AFTER_FRAMES=180 dotnet run --project experiments/e06-camera-and-collision/E06CameraAndCollision.csproj --no-restore
env E07_SMOKE_EXIT_AFTER_FRAMES=170 dotnet run --project experiments/e07-animation/E07Animation.csproj --no-restore
```

## Observe

The camera/collision smoke includes:

```text
Collision: aabb=True, circle=True.
Smoke: exit.
```

The animation smoke logs:

```text
Transition: Idle -> Walk
Transition: Idle -> Jump
Smoke: exit.
```

## Expected Visual State

Camera/collision smoke logs AABB and circle collision; animation smoke logs idle/walk/jump transitions.

## Walkthrough

MonoGame gives you drawing primitives and matrices; it does not provide a built-in 2D camera, collision system, or animation graph.

`Camera2D` keeps world-space position and zoom while drawing passes a transform matrix to `SpriteBatch.Begin`. The GUI smoke setting `E06_SMOKE_EXIT_AFTER_FRAMES=180` pans, zooms, and moves shapes on fixed frame ranges, so the log can prove the same AABB and circle collision path that the window draws in red or green.

`Collision2D` contains deterministic AABB and circle overlap tests. Keeping that math outside `Game1` lets the test project verify collision behavior without rendering, while `CollisionScene` supplies the moving and static shapes that `Game1` draws.

Animation is regular game code: `FrameAnimation` advances frames, `CharacterStateMachine` emits idle, walk, and jump transitions, and `AnimationController` maps character state to animation frames. The GUI smoke setting `E07_SMOKE_EXIT_AFTER_FRAMES=170` feeds a deterministic input sequence so the transition log and the drawn animation agree.

## Common Failures

- Collision debug drawing is user code; MonoGame does not provide these outlines automatically.
- Animation graphs are not built in. State transitions, frame timing, and reset behavior are your responsibility.
- Smoke mode replaces manual input so camera, collision, and animation can be checked without key events.
