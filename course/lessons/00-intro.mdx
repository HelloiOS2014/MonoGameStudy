## Goal

Understand how the completed experiments and integrated demo form the MonoGame study path. You will inspect existing finished code rather than build a new game from a blank folder.

## Run

Run the playable demo manually:

```bash
dotnet run --project demo/integrated-demo/IntegratedDemo.csproj
```

Run the same demo in automated smoke mode:

```bash
env DEMO_SMOKE_EXIT_AFTER_FRAMES=160 dotnet run --project demo/integrated-demo/IntegratedDemo.csproj --no-restore
```

## Observe

Manual mode opens a MonoGame window. Press `Enter` to start, collect the yellow pickups, avoid the red hazards, then restart from the win or loss screen.

Smoke mode exits by itself. Near the end of stdout, look for:

```text
Phase: started.
Phase: won.
Phase: restarted.
Smoke: exit.
```

## Expected Visual State

Integrated demo manual mode opens a small collector arena; smoke mode reaches started, won, restarted, and exit logs.

## Walkthrough

The repo has two main learning areas. `experiments/` contains topic-focused finished states for the game loop, rendering, input, audio, content loading, collision, animation, and publishing. `demo/integrated-demo/` is the capstone that combines the useful parts into a small 2D collector loop.

Use the experiments when you want to isolate one subsystem. Use the integrated demo when you want to see the pieces fit together. The demo remains capstone evidence for this learning framework, not a production game expansion target.

Most runtime lessons follow the same pattern: `Game1` owns MonoGame lifecycle wiring, while deterministic state is moved into smaller classes where tests can run without a graphics device. Smoke settings make GUI experiments report repeatable command evidence instead of requiring manual screenshots for every checkpoint.

## Common Failures

- This tutorial assumes macOS DesktopGL. Other platforms may work, but they were not the Phase 1 target.
- The first restore or build can take minutes because NuGet packages, MonoGame tools, and generated content may need to be prepared.
- A normal `dotnet run` opens a GUI window. Use the smoke command for a non-interactive confidence check.
